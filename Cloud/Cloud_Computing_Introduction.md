# Cloud Computing Introduction

## PC Centric vs Document-Centric

Unlike traditional computing, the cloud computing model isn't PC-centric—it's **document-centric**. Which PC you use to access a document simply isn't important.

---

## What Cloud Computing is?

First, cloud computing isn't network computing:
- **Network computing**: Applications and documents are hosted on a single company's server and accessed over the company's network
- **Cloud computing**: It's much bigger than that. It encompasses multiple companies, multiple servers, and multiple networks

Additionally, unlike network computing:
- Cloud services and storage are accessible from **anywhere in the world** over an internet connection
- Network computing access is limited to the company's network only

### Key Properties of Cloud Computing

Cloud computing has **six main key properties**:

1. **User-centric** — Centered around users' needs
2. **Task-centric** — Focused on completing tasks
3. **Powerful** — Offers immense computing resources
4. **Accessible** — Available from anywhere
5. **Intelligent** — Smart resource management
6. **Programmable** — Can be customized and automated

---

## P2P (Peer-to-Peer) Computing

### Definition and Characteristics

P2P computing defines a network architecture in which each computer has **equivalent capabilities and responsibilities**.

#### Key Concepts

**P2P as an Equalizing Concept:**
- In the P2P environment, every computer is both a client and a server
- There are no masters and slaves
- P2P enables direct exchange of resources and services
- There is no need for a central server—any computer can function as one when needed

**P2P as a Decentralizing Concept:**
- Control is decentralized, with all computers functioning as equals
- Content is dispersed among various peer computers
- No centralized server hosts the available resources and services

### The Internet as P2P Implementation

Perhaps the most notable implementation of P2P computing is the **Internet** itself. Let's explore its early days and evolution!

#### ARPAnet: The Grandparent of the Internet

**What was ARPAnet?**
- **ARPAnet** stands for Advanced Research Projects Agency Network
- It was created by the U.S. Department of Defense
- It was the grandparent of today's Internet

**How ARPAnet Worked:**
- All computers connected to the network were like "best friends who shared everything"
- Computers weren't split into "servers" (give information) and "clients" (receive information)
- Instead, **every computer could both give and receive information equally** — a true peer-to-peer system

**A Helpful Analogy:**
Think of a group of friends sitting in a circle, each with a different book:
- Anyone can ask to borrow a book
- Anyone can lend their book out
- This is how ARPAnet worked with computer resources—everyone could share what they had and use what others shared

#### How the Internet Has Changed

**Then vs. Now:**
- **Then (ARPAnet era)**: Computers shared resources equally in a peer-to-peer manner
- **Now**: We typically connect to big servers (like Google or Facebook) to get information or use services—more like going to a library to borrow books rather than sharing with friends

**Why This Matters:**
- Many people today don't know about this original design of the Internet
- It shows how technology and its uses can change over time
- The Internet evolved from a way for research sites to share resources equally into a global network used by billions of people in many different ways

**Future Possibilities:**
- Understanding this history helps us appreciate how the Internet has evolved
- It gives us ideas about how it could change in the future
- Perhaps someday we'll see a return to more peer-to-peer systems in some areas of the Internet!

## Usenet: An Early P2P Success Story

### What is Usenet?

**Usenet** was an early form of online communication that worked like a giant bulletin board system—existing before the World Wide Web as we know it today.

### How Usenet Worked

**Peer-to-Peer (P2P) Network:**
- Computers talked directly to each other, rather than going through a central server
- This is what we call "peer-to-peer" or P2P

**Message Sharing:**
- When someone posted a message on Usenet, it would spread to all other computers in the network
- It's like telling a secret to your friends, who tell their friends, until everyone knew

**Distributed Content:**
- Every computer in the Usenet network held a copy of all the messages
- You could connect to any computer in the network and see all the messages, regardless of which computer they were originally posted on

**Client/Server + P2P Hybrid Model:**
- As a user, you'd connect to one specific computer (server) — this part worked like a traditional client/server model
- Behind the scenes, all the Usenet servers were talking to each other in a P2P fashion

### Usenet and Cloud Computing: The Connection

How does Usenet relate to modern cloud computing? The key is in how information was distributed and accessed:

**Distributed Data:**
- Just like Usenet spread messages across multiple computers, cloud computing often involves storing data across multiple servers

**Access from Anywhere:**
- With Usenet, you could access all messages from any server
- Similarly, cloud computing allows you to access your data from any device with an internet connection

**Redundancy:**
- If one Usenet server went down, the messages were still available on others
- Cloud computing uses similar principles to ensure your data is always available

### Important Note

While Usenet and modern cloud computing are quite different in many ways, Usenet demonstrated early principles that would later become important in cloud computing:
- Distributed systems
- Data replication
- Universal access

**However**, Usenet was a specific application, while cloud computing is a broad concept encompassing many technologies and services. The comparison is meant to show how early Internet technologies had features that foreshadowed aspects of modern cloud computing.

## Distributed Computing

### Understanding the Concept

**Distributed computing** is a key part of the peer-to-peer (P2P) model in cloud computing.

### Breaking It Down

**P2P Model Analogy:**
Imagine a group of friends sharing music files directly with each other, without using a central server. That's the essence of P2P—a network where each computer can act as both a client and a server, sharing and receiving resources directly.

**Distributed Computing Analogy:**
Think of a huge puzzle that's too big for one person to solve quickly. Distributed computing is like dividing that puzzle among many people, each working on their piece simultaneously. In computer terms, it's about **splitting a big computing task across multiple connected devices**.

### P2P Model and Distributed Computing

When we say that distributed computing is an important subset of the P2P model, we mean that **one of the key ways P2P networks are used is for sharing computing power to tackle big problems**.

### Real-World Example: Multiplayer Games

Imagine a massive online multiplayer game:
- The game doesn't run on one big central computer
- Each player's computer contributes some processing power to keep the game world running smoothly
- **That's distributed computing in action within a P2P model**

### Why This Matters for Cloud Computing

This concept is crucial in cloud computing because it allows for:
- **More efficient use of resources**: Instead of relying on one powerful central computer, tasks are spread across many devices
- **Increased speed**: Potential for faster processing
- **Greater capacity**: Increased overall computing power

### Summary of Distributed Computing

Distributed computing is a key application of the peer-to-peer model, where computing tasks are shared across multiple devices in a network, leading to more efficient use of resources in cloud computing environments.

## How Cloud Computing Works

### The Cloud Metaphor

Imagine the Internet as a **giant, invisible cloud** that floats all around us. This "cloud" is actually made up of:
- Countless connected computers and servers
- Each with its own unique address (called an **IP address**)

### What Cloud Computing Enables

When we talk about cloud computing, we're referring to **using the resources of this vast network** to do things that would be difficult or impossible with just one computer.

### What Makes Cloud Computing Special

**Massive Computing Power:**
- Instead of relying on your own computer's processor, you can tap into the combined power of many computers in the cloud
- It's like having a supercomputer at your fingertips!

**Huge Storage Capacity:**
- Think of the cloud as an endless digital storage locker
- You can save almost unlimited amounts of data without worrying about running out of space on your own device

**Collaboration on a Grand Scale:**
- The cloud makes it easy for people all over the world to work together on the same projects
- Share files instantly and communicate in real-time

### A Practical Example

Imagine you and your classmates are working on a big group project:
- **Without cloud**: Pass around USB drives or email files back and forth
- **With cloud**: All access and edit the same documents simultaneously through a cloud-based service like Google Docs
- **Result**: The power of cloud collaboration!

### Key Advantages

The beauty of cloud computing is that it's **accessible from anywhere with an internet connection**:
- Use a smartphone, tablet, or computer
- Tap into powerful resources seamlessly
- Work across different devices effortlessly

### Important Considerations

While the cloud offers many advantages, it also comes with important considerations:
- **Data security**: Protect your information
- **Privacy**: Be mindful of what you're storing
- **Best practices**: Use strong passwords to protect your accounts

### Summary

**Cloud computing harnesses the vast network of the Internet to provide:**
- Immense computing power
- Storage capabilities
- Collaboration tools

These extend far beyond what a single computer can offer, changing the way we work, learn, and interact in our increasingly connected world.

## 1. SaaS (Software as a Service):

Let's break down Software as a Service (SaaS) in a way that's easy to understand, using Microsoft/Azure-related examples:

Imagine a giant library in the cloud where instead of books, there are computer programs. SaaS is like checking out these programs whenever you need them, without having to buy and install them on your own computer.

---

### Here's how it works:

- **The Program Lives Online:** Instead of installing software on your computer, it's hosted on the cloud by providers like :contentReference[oaicite:0]{index=0} and accessed via browser or app.

- **Pay-as-you-go:** Rather than buying the software outright, you pay for using it (subscription model), like :contentReference[oaicite:1]{index=1}.

- **Access Anywhere:** You can use the software from any device with internet access—laptop, mobile, tablet.

- **One Program, Many Users:** The same application is shared across thousands/millions of users securely.

- **Web API:** You interact with the program through something called a Web API. Think of this as the library's catalog system - it helps you find and use what you need.

---

### The big advantage of SaaS:

The big advantage of SaaS is that you don't need to worry about maintaining or updating the software. The service provider (like Microsoft) handles updates, scaling, and security.

---

### Some real-world examples of SaaS include:

- :contentReference[oaicite:2]{index=2} (Word, Excel, Outlook online)  
- :contentReference[oaicite:3]{index=3} for storing and sharing files  
- :contentReference[oaicite:4]{index=4} for communication and meetings  

---

### One potential challenge with SaaS:

One potential challenge with SaaS is that you need a reliable internet connection. Also, you're relying on the service provider for:
- Data security  
- Availability  
- Performance  

---

### Summary:

In summary, SaaS is a way of delivering software over the internet, where users pay for access rather than ownership, and can use the software from anywhere with an internet connection. It's a key part of how cloud computing is changing the way we work and collaborate online.

## 2. PaaS (Platform as a Service)

Let's break down Platform as a Service (PaaS) from a **developer’s perspective**, especially using Azure:

Imagine you want to build an application, but instead of setting up servers, installing software, and managing infrastructure, you are given a **ready-to-use development platform**. You just bring your code and deploy it.

In this case, the cloud provider gives you a **platform** where everything is pre-configured:
- Operating System
- Runtime (Java, Node.js, .NET)
- Web server
- Scaling and load balancing

---

### 🔹 How it works:

- **You write the application code**
- **You deploy it to the platform**
- The cloud handles:
  - Infrastructure
  - Runtime setup
  - Scaling
  - Maintenance

---

### 🔹 Azure Example:

- Azure App Service → deploy Spring Boot / Node apps directly  
- Azure SQL Database → managed database  
- Azure Blob Storage → file storage  

👉 You don’t install Java or configure servers manually.

---

### 🔹 How Platform is Preconfigured (Very Important)

When using Azure App Service, you don’t build the environment from scratch. Instead, you **select a pre-configured runtime stack** provided by the platform.

#### ✔ While creating App Service, you choose:

- Runtime stack → Java / Node.js / Python / .NET  
- Version → Java 11 / 17, Node 18, etc.  
- OS → Linux / Windows  

👉 Example (Spring Boot):
- Runtime → Java  
- Version → Java 17  

---

#### 🔧 What Azure does internally:

Once you select the stack, Azure automatically:

- Sets up the Operating System  
- Installs the selected runtime (e.g., Java 17)  
- Configures web server / execution environment  
- Prepares deployment support  

👉 You **do NOT**:
- Install Java manually  
- Configure Tomcat or server  
- Handle OS setup  

---

#### ⚠️ Important Limitation (Stack Binding)

Because the platform is pre-configured:

- If you select **Java runtime** → you can deploy only Java apps  
- You **cannot directly deploy Python apps** in that setup  

---

#### 🔥 Exception (Using Custom Container (Still PaaS))

If you use a **custom Docker container**:

- You define your own runtime  
- You can run any language (Python, Go, Node, etc.)

👉 So:

- Built-in runtime → **restricted to selected stack**  
- Container-based → **no restriction**

---

#### 🎯 One-Line Understanding

> In PaaS, the platform is pre-configured with a specific runtime stack, and developers select the environment instead of setting it up manually.

---

### 🔹 Real Development Flow:

1. Develop application locally  
2. Push code to GitHub / Azure DevOps  
3. Deploy to Azure App Service  
4. Connect with Azure SQL / Blob  

---

### 🔹 Advantage:

- Faster development  
- No infrastructure management  
- Built-in scaling  

---

### 🔹 Limitation:

- Limited control over environment  
- Cannot customize OS or low-level configs  

---

### 🔹 One-Line Summary:

> PaaS provides a ready-to-use platform where developers focus on code, and the cloud manages the infrastructure.

---

## 3. IaaS (Infrastructure as a Service)

Now let’s understand IaaS in a practical way:

Imagine you are given a **virtual computer (VM)** in the cloud. You have full control, just like your own system.

---

### 🔹 How it works:

- Cloud provider gives:
  - Virtual Machine
  - Storage
  - Network

- You are responsible for:
  - OS setup
  - Runtime installation (Java, Node)
  - Server configuration (Tomcat, Nginx)
  - Deployment
  - Scaling
  - Monitoring

---

### 🔹 Azure Example:

- Azure Virtual Machines  

---

### 🔹 Real Development Flow:

1. Create VM  
2. Install OS updates  
3. Install Java  
4. Configure Tomcat  
5. Deploy application  
6. Set up scaling and monitoring  

---

### 🔹 Advantage:

- Full control  
- Flexible environment  

---

### 🔹 Limitation:

- High maintenance  
- More DevOps effort  

---

### 🔹 One-Line Summary:

> IaaS provides infrastructure where developers manage everything except physical hardware.

---

## 4. CaaS (Container as a Service)

Now let’s connect this with different situation (very important):

You are using:
- Azure Red Hat OpenShift (ARO)
- Containers

👉 This is CaaS

---

### 🔹 What is CaaS?

CaaS provides a platform to **deploy and manage containers**, without managing the underlying infrastructure fully.

---

### 🔹 How it works:

- You:
  - Create Docker containers  
  - Define deployments (YAML)  
  - Manage application scaling  

- Cloud provider:
  - Manages Kubernetes cluster  
  - Handles nodes and infrastructure  

---

### 🔹 Azure Example:

- Azure Red Hat OpenShift (ARO)  
- Azure Kubernetes Service (AKS)  

---

### 🔹 Real Development Flow:

1. Develop application locally  
2. Create Docker image  
3. Push image to registry  
4. Deploy to ARO/AKS using YAML  
5. Manage pods, services, scaling  

---

### 🔹 Advantage:

- High scalability  
- Microservices friendly  
- More control than PaaS  

---

### 🔹 Limitation:

- More complexity than PaaS  
- Need Kubernetes knowledge  

---

### 🔹 One-Line Summary:

> CaaS allows developers to deploy containerized applications while the cloud manages the underlying infrastructure.

---
